Testing Pyramid

Testing Tools

Typical Testing Tools:

Jest, Testing Library, Vitest, Playwright, Istanbul, Cypress, Webdriver.io, Selenium, happy-dom

Why it's a pyramid?

Tests at the bottom are cheap and fast - we can and should create a lot of them

Often tradeoffs: precision vs velocity

The more we move up, the more expensive and slow they become

Shift-Lefting

What is shift-lefting and why should we do this?

Shift-lefting - doing or considering things earlier in the process, not in the end

"A good plan is half the battle"

Advantages of shift-left approach

  • Identify issues/bugs early
  • Reduce development costs
  • Improve product quality
  • Prevent technical debt ("we will fix it later")

Visual Regression Tests

Visual regression tests

Oftentimes underestimated

They give you the confidence you need to ship your code regularly without fear (CI/CD, continuous delivery)

regression tests = should prevent you from accidently introducing bugs in your code

theoreticall all tests are kind of regression tests - if they turn from red to green

but TRUE regression tests tell you not only that something is wrong, but also what exactly

regression tests compare in an onbious-way some old state with new state

something you know and which is valid is called = baseline

they can help greatly reduce the amount of your tests and assertions

"A picture says more than a thousand words"

two types: visual and DOM

The workflow

Putting in all together

  1. Align with your designers early. Decide on the so called component API: Component Name, property names, property values, default values, prop types
  2. Try to also align on stories you will have on both sides
  3. Wait for the design to be finalized
  4. Implent the component: Use storybook testing suite to create a bullet proof implementation: unit tests, ally automatically, interaction tests
  5. Let Chromatic run visual regression tests and review the results
  6. Ship your code with confidence and sleep well at night :)
